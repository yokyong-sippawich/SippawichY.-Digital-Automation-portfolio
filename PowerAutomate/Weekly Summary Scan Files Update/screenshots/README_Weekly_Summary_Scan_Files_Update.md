# Weekly Summary – Scan Files Update (Power Automate Flow)

This flow generates a weekly email summarizing scanned files for preventive and corrective maintenance work orders. It runs an Office Script on a SharePoint‑hosted Excel file to extract last week’s data (Fri → Thu), formats the results into an HTML table, and sends the summary to stakeholders. If no data is found, the flow sends a short “no data” notification.

---

## 📌 Overview

- **Schedule:** Every Wednesday (UTC+07:00, Bangkok)
- **Systems used:** Power Automate, SharePoint (Documents), Excel Online (Office Scripts), Outlook
- **Purpose**
  - Automate weekly summary of scanned Excel files
  - Convert extracted rows to a clean HTML table
  - Send a formatted email to stakeholders
  - Send a “no data” email when no rows are returned

---

## 🧭 High‑level Flow

1. **Recurrence** → runs weekly on Wednesday  
2. **Run script (Excel Online)** → `ReadSheets_LastFriToThu` on `SCAN RECORD.xlsx`  
3. **Condition:** check if `rows > 0`  
   - **Yes (rows > 0)**  
     a) Create **HTML table** from `result.rows`  
     b) Compose **IncludedSheetsString** (comma‑separated)  
     c) Compose **EmailBody** (HTML template)  
     d) **Send summary email**  
   - **No (rows = 0)**  
     a) **Send no‑data email**

## ⚙️ Key Parameters & Expressions

### 1) Recurrence
- Frequency: `Week`  
- Day: `Wednesday`  
- Time zone: `UTC+07:00 (Bangkok, Hanoi, Jakarta)`  
- Set your preferred hour/minute in “At these hours / At these minutes”.

### 2) Run script (Excel Online)
- Location: `SharePoint Site – PY SER`  
- Document Library: `Documents`  
- File path:  
  `/08. PI & Mobilisation/57. Tracking BTSC Notification and Service Order/1. PK Plant/SCAN RECORD.xlsx`
- Script name: `ReadSheets_LastFriToThu`
- Expected outputs (as in your flow):
  - `body('Run_script')?['result']?['rows']` → array of data rows  
  - `body('Run_script')?['result']?['includedSheets']` → array of sheet names  
  - *(optional)* `window.start`, `window.end` → for subject range

### 3) Condition (has data?)
Use this expression (true = has data):
```text
greater(length(body('Run_script')?['result']?['rows']), 0)
```

### 4) Create HTML table (when rows > 0)
- **From:** `result.rows`  
- **Columns:** Custom  
  - `Work order No.` → **Work order No.**  
  - `Description` → **Description**  
  - `Functional Location` → **Functional Location**  
  - `SourceSuffix` → *(fx) your formula*

### 5) IncludedSheetsString (Compose)
```text
join(coalesce(body('Run_script')?['result']?['includedSheets'], createArray()), ', ')
```

### 6) EmailBody (Compose) — HTML template  
Add a **Compose** action named `EmailBody` and paste this expression:
```text
@{concat(
'<html><head><meta charset="utf-8"/>',
'<style>',
'table{border-collapse:collapse;width:100%;}',
'th,td{border:1px solid #b0c4de;padding:6px 8px;text-align:left;}',
'th{background:#e6f0ff;}',
'p{margin:6px 0;}',
'</style></head><body>',
'<h2 style="margin:6px 0;">Weekly Summary Scan Files Update</h2>',
'<div style="font-size:13px;color:#466;">Included sheets: ',
string(outputs('IncludedSheetsString')),
'</div>',
'<div style="margin:10px 0;">',
outputs('Create_HTML_table'),
'</div>',
'<p style="margin-top:10px;">This email was automatically generated by Microsoft Power Automate.</p>',
'<p>If you have any questions, please contact me.</p>',
'<p>Best regards,<br/>Automation Bot</p>',
'</body></html>'
)}
```

### 7) Send an email (V2)
- **To:** Stakeholders (per your flow)  
- **Subject (examples):**
  - If the script returns the date range:
    ```text
    Weekly Summary Scan Files Update (Between @{body('Run_script')?['result']?['window.start']} to @{body('Run_script')?['result']?['window.end']})
    ```
  - If not, use a fixed subject or compute Fri→Thu with `addDays()` + `formatDateTime()`.
- **Body:** `outputs('EmailBody')`  
- **Importance:** Normal

## ✅ Notes & Tips

- After importing to another environment, fix **Connection references** (SharePoint, Excel Online, Outlook).  
- Ensure the Excel file path matches exactly.  
- If your current flow has True/False branches reversed, align them so:
  - **True = has data** → build table + send summary  
  - **False = no data** → send no‑data email
